# Setting up Certificate Services

There's one more thing we need to do and that is to set up certificate services. So click on **Manage > Add Roles and Features**:

<center> <img src="/44.png" width="300" height="600" class="center" /> </center>

Click **Next** on the first screen, then select **Role-Based or feature-based installation**:

<center> <img src="/45.png" width="700" height="600" class="center" /> </center>

Click **Next** again, until you're on the Server Roles screen. Then make sure to tick **Active Directory Certificate Services**:

<center> <img src="/46.png" width="700" height="600" class="center" /> </center>

Click **Add Features** and then **Next** until you reach the **Role Services** Screen. Make sure the **Certificate Authority** option is ticked.

<center> <img src="/47.png" width="700" height="600" class="center" /> </center>

Tick the restart option:

<center> <img src="/48.png" width="700" height="600" class="center" /> </center>

Once the installation is done, click **Configure AD CS on the destination server**:

<center> <img src="/49.png" width="700" height="600" class="center" /> </center>

Hit **Next** and then tick **Certificate Authority**:

<center> <img src="/50.png" width="700" height="600" class="center" /> </center>

Keep **Enterprise CA** checked, Root Ca checked, Create a new private key also checked.

On the Cryptography section, SHA256 is fine:

<center> <img src="/51.png" width="700" height="600" class="center" /> </center>

Leave the CA Names as same, set the validity period to 99 years (because we want to have this lab long term):

<center> <img src="/52.png" width="700" height="600" class="center" /> </center>

Keep clicking **Next** until you're done. Then restart your machine.

.. and just like that, your domain controller is set up. In the next section, we'll set up our user machines.
